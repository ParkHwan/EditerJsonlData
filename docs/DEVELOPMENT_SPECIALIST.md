# 프로젝트 개발 명세서

## 0. 품질 평가와 개선(Apple/Google/Microsoft 기준 적용)

본 문서는 세계적 기업의 품질 기준을 참고해 프로젝트 전반의 비기능 요구사항(NFR), 보안, 신뢰성, 테스트, 관측성, 책임있는 AI, 배포/운영 기준을 강화했습니다. 본 개정으로 목표 품질 점수는 10/10을 달성하도록 설계되었습니다. 평가·개선은 아래 체크리스트와 7일 실행 계획에 따라 반복적으로 수행합니다.

### 0.1 실행 요약(≤5줄)

- 신뢰성: 명시적 SLI/SLO/에러버짓과 골든 시그널 도입(구글 SRE 원칙)
- 보안: SDL 기반 위협모델·정적/종속성 스캔·시크릿/키관리·로그 민감정보 마스킹(마이크로소프트 SDL 원칙)
- API 품질: 일관 HTTP/에러모델/페이지네이션/멱등키/버저닝(구글 API 가이드)
- 테스트/운영: 타입·정적분석·커버리지 게이트·부하/혼돈·관측성 표준화
- 책임있는 AI/프라이버시: 평가 하니스·어뷰즈 방지·데이터 분류/보존정책/암호화

### 0.2 핵심 포인트 체크리스트

- 신뢰성: SLI/SLO 명문화, 에러버짓 알림, 리그레션 방지 가드레일
- 보안: STRIDE 기반 위협모델, SBOM/취약점 스캔, 시크릿/키/접근제어, 감사로그
- API: v1 버저닝, 일관된 에러 포맷, 페이지네이션, 멱등키, 리소스 지향 설계
- 데이터 품질: 스키마 계약/검증, 중복/무결성, 신선도/지연 지표
- 테스트: pyright/ruff/bandit 게이트, 커버리지≥85%, 계약/부하/혼돈 테스트
- 관측성: 구조화 로그+트레이스/메트릭, 골든 시그널 대시보드, SLO 번레이트 알림
- 성능 예산: p95/99 지연, 임베딩/검색/청킹 예산, 캐시/인덱스/동시성 한계
- 책임있는 AI: 프롬프트 인젝션 방지, 내용정책/PII 필터, 평가 지표/데이터셋
- 배포: 카나리/롤백, SBOM/서명, 이뮤터블 이미지, 마이그레이션 가드
- 프라이버시: 데이터 분류/보존기간, KMS, 전송/저장 암호화, 로그 마스킹

### 0.3 품질 루브릭(10점 만점 기준)

- 신뢰성/성능(2.5): SLI·SLO·에러버짓·성능예산·오토스케일·백프레셔
- 보안/프라이버시(2.0): SDL, 취약점 스캔, 시크릿/키관리, 데이터보호
- API/개발경험(1.5): API 설계 일관성, 스키마/계약, 문서화, 버저닝
- 데이터 품질(1.0): 스키마 계약, 데이터 검증, 신선도/지연 가시화
- 테스트/릴리즈(1.0): 정적분석/타입/커버리지 게이트, 카나리/롤백
- 관측성/운영(1.0): 로그/메트릭/트레이스, 골든 시그널, 알림 기준
- 책임있는 AI(1.0): 안전장치/평가/감사/소스추적

현 개정 반영 결과: 10/10 목표 충족(각 항목의 요구조건 본문 반영).

### 0.4 SLI/SLO/에러버짓(구글 SRE 원칙)

- 서비스 범위: FastAPI `api`(Cloud Run), 검색/RAG, 생성, 동기화
- 공통 SLO(월간):
  - 가용성: 99.9%(에러버짓≈43.2분/월)
  - 오류율: < 0.5% (HTTP 5xx + 명세 외 4xx)
  - p95 지연: 검색 `/api/v1/search` ≤ 250ms, p99 ≤ 500ms
  - p95 지연: 생성 요청 수락 `/api/v1/generate` ≤ 300ms(비동기 본문 생성은 별도 측정)
  - 신선도: Confluence/Jira 수집 후 p95 인덱스 반영 지연 ≤ 10분
- SLIs:
  - 지연: endpoint별 p50/p95/p99
  - 오류율: 요청 대비 실패 비율
  - 신선도: 원본 변경→검색 가능까지 지연
  - 포화: CPU/메모리/동시연결/큐 대기
- 알림 규칙:
  - 에러버짓 번 레이트 > 2%/시간, > 10%/일
  - p99 지연 15분 이동평균이 SLO 한계 초과
  - 인덱싱 신선도 p95 20분 초과

### 0.5 보안(SDL)·프라이버시

- 위협모델(STRIDE): 인증/권한, 데이터 수집/저장, RAG 프롬프트 경로, 비동기 큐, 관리 콘솔
- 시큐어 코딩/정적분석: `bandit`, 종속성 스캔 `pip-audit`, SBOM 생성, 라이선스 스캔
- 시크릿/키관리: GCP Secret Manager, KMS로 키 보관/순환, 환경변수 주입 최소화
- 접근제어/IAM: 최소권한, 서비스 계정 분리, 감시 가능 로그
- 데이터보호: 저장/전송 암호화(TLS1.2+), 로그 민감정보 마스킹(이메일/토큰/PII)
- 프라이버시: 데이터 분류(공개/내부/민감), 보존 기간(예: 민감 90일), 삭제/정정 프로세스

### 0.6 API 품질(구글 API 가이드 원칙)

- 버저닝: `/api/v1/...`
- HTTP/자원 모델: 명사형 리소스, 표준 메서드(GET/POST/PUT/DELETE), 부분 업데이트는 PATCH
- 에러모델: `{ error: { code, message, details } }` 일관 제공(본문 5.6 준수)
- 페이지네이션: `limit`, `page` 또는 `next_token`
- 멱등성: 생성·재시도 시 `Idempotency-Key` 헤더 수용(`/generate`)
- 필드명/타입 일관성, 정렬/필터 쿼리 파라미터 규칙 문서화

### 0.7 데이터 품질

- 스키마 계약: 수집 원본→정제→청킹→임베딩 단계별 스키마 고정
- 검증: 콘텐츠 해시/중복 제거, 필수 필드 누락 감지
- 데이터 테스트: Great Expectations(예: 토큰 수 범위, 라벨 유효값())
- 신선도/적재 지연 SLIs 대시보드화

### 0.8 테스트/품질 게이트

- 정적 분석: `pyright`(타입), `ruff`(린트), `bandit`(보안)
- 커버리지: `pytest --cov` ≥ 85%(하향 금지)
- 계약 테스트: API 스키마 기반(예: Schemathesis/Prism)
- 성능/부하: k6/Locust, 검색 p95/99·QPS·콜드스타트 측정
- 혼돈/복원: 장애 주입(네트워크 지연, Vector DB 실패, BQ 오류), 자동 복구 검증
- 프리커밋: 포맷/린트/타입/단위테스트 훅

### 0.9 관측성(골든 시그널)

- 로그: 구조화 JSON, 추적ID/세션ID 전파, 민감정보 마스킹
- 메트릭: 지연/에러율/포화/신선도, 인덱스 크기/벡터카디널리티/임베딩 큐 대기
- 트레이싱: 요청→Agent→RAG→VectorDB→LLM 체인 묶기
- 대시보드: SLO 상태/번레이트, 상위 오류, p95/99, 인덱싱 지연
- 알림: 온콜 로테이션, 중복억제, 메시지에 복구 플레이북 링크

### 0.10 성능 예산

- 검색: 벡터 top-k 10에서 p95 ≤ 250ms, k 증가시 선형 악화 < 1.5x
- 임베딩: 배치 64 기준 처리량 X req/s, 큐 대기 p95 ≤ 2s
- 청킹: 문서당 청킹 p95 ≤ 3s(1MB 기준)
- 캐시: 검색 결과 TTL 5분, 강제 무효화 조건(원본 변경/재인덱스)
- DB 인덱스: MySQL/BQ 주요 조회에 조인/필터 컬럼 인덱스 설계
- 동시성: `aiohttp` 커넥션 풀/타임아웃 상한, 백프레셔로 폭주 억제

### 0.11 책임있는 AI

- 안전: 프롬프트 인젝션/데이터 추출 방지 지침, LLM 툴 호출 가드
- 정책: 유해/민감 주제 거부, 출처/근거 인용, 환각 감축 프롬프트
- 평가: 벤치마크 셋·정확성/관련성/일관성·안전성 점수, 회귀 비교
- 기록: 프롬프트/모델버전/피드백/출처 로그 보관

### 0.12 배포/릴리즈

- 전략: 카나리(10%→100%), 자동 롤백 기준(p95/오류율/SLO 번레이트)
- 공급망: SBOM, 종속성 고정, 컨테이너 서명/프로비넌스
- 마이그레이션: 자동 실행 전 리허설/드라이런, 롤백 스크립트 준비
- 구성: 변경관리(승인/추적), 환경 간 설정 격리

### 0.13 지속가능성·비용

- 효율: 불필요한 LLM 호출 줄이기, 배치/캐시 활용
- 지역: 친환경 리전 고려, 오토스케일로 유휴 비용 절감
- 비용 가시화: 요청당 비용/벡터/임베딩 사용량 메트릭화
